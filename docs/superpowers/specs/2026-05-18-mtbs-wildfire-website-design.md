# MTBS Wildfire Website — Design Spec
**Date:** 2026-05-18  
**Status:** Approved

---

## Overview

A single combined website to showcase the MTBS Wildfires project — its process, analysis, and results. Built with Streamlit (Python) and targeted at two audiences: personal portfolio reviewers and academic/research peers.

The site uses a **3-tab structure**: Story · Research · Explorer. The Story tab uses custom HTML components for cinematic narrative effect; Research and Explorer use pure Streamlit. The visual theme is **Dark Fire** — dark backgrounds with ember orange/red accents throughout.

---

## Audiences

| Audience | Goal |
|---|---|
| Portfolio reviewers | See technical depth, communication ability, and end-to-end project ownership |
| Academic / researchers | See methodology, data sources, reproducibility, and key findings |

---

## Tech Stack

| Layer | Choice | Rationale |
|---|---|---|
| Framework | Streamlit | Python-native, fast iteration, deploys free to Streamlit Cloud |
| Story tab rendering | `st.components.v1.html()` | Full visual/design control for narrative sections |
| Charts | Plotly | Interactive, dark-theme-compatible, integrates natively with Streamlit |
| Maps | Folium + streamlit-folium | Interactive fire event map with click-to-inspect |
| Deployment | Streamlit Cloud | Free, one-click deploy from GitHub |
| Runtime data | Pre-processed CSV / GeoJSON | No live GEE calls at runtime — keeps site fast and auth-free |

## Architecture

### File Structure

```
mtbs_wildfires/
├── app.py                     # Entry point: tab switcher, global dark CSS theme
├── pages/
│   ├── story.py               # Injects HTML components for narrative sections
│   ├── research.py            # Pure Streamlit: methods, results, report links
│   └── explorer.py            # Pure Streamlit: Folium map + filters + detail panel
├── components/
│   ├── story_hero.html        # Hero section HTML template
│   └── story_sections.html    # Narrative section HTML templates
├── data/
│   ├── mtbs_ca_summary.csv    # Pre-processed fire event summary (from notebooks)
│   └── evi_exports/           # Static EVI map images exported from evi.ipynb
└── src/                       # Existing notebooks + utils (unchanged)
```

### Data Flow

```
GEE / AWS  →  Notebooks (existing)  →  CSV / GeoJSON  →  Streamlit app
```

Pre-processing happens offline in the existing notebooks. The website reads static output files — no Google Earth Engine authentication required at runtime.

### Required Pre-processed Data

`data/mtbs_ca_summary.csv` must contain at minimum:

| Column | Description |
|---|---|
| `event_id` | MTBS event ID (e.g. `CA3983912034520210702`) |
| `event_name` | Human-readable fire name |
| `ig_date` | Ignition date (YYYY-MM-DD) |
| `year` | Ignition year (integer) |
| `latitude` | Centroid latitude |
| `longitude` | Centroid longitude |
| `acres` | Total acres burned |
| `severity` | Severity tier: Low / Medium / High |
| `max_temp_c` | Max temperature (°C) during event from Open-Meteo |
| `min_humidity_pct` | Min relative humidity (%) during event from Open-Meteo |

This file is generated by running `mtbs_source_coop.ipynb` and `openmeteo_example.ipynb` and joining the outputs.

`data/evi_exports/` contains PNG images of EVI maps exported from `evi.ipynb` for a representative set of events, named `{event_id}.png`.

## Tab 1 — Story

**Implementation:** `pages/story.py` using `st.components.v1.html()` to embed hand-crafted HTML/CSS narrative sections. Data values (stat counters, chart data) are injected from Python into the HTML templates at render time.

### Sections (top → bottom)

1. **Hero** — full-width dark fire gradient background, title "40 Years of Fire", California + date range subtitle, scroll prompt
2. **Stat counters** — 4 key numbers in a horizontal bar: total acres burned, fire event count, % high-severity events, time span. All values computed at runtime from `mtbs_ca_summary.csv`.
3. **The Trend** — narrative paragraph about escalating fire severity + Plotly bar chart of acres burned per year (1984–2022)
4. **Vegetation Recovery** — narrative explaining EVI analysis + 3-column visual comparing pre-fire / post-fire / 1yr recovery EVI colour gradients
5. **CTA** — buttons linking to Explorer tab and Research tab

---

## Tab 2 — Research

**Implementation:** `pages/research.py` using pure Streamlit components.

### Sections (top → bottom)

1. **About** — project summary paragraph, View Report PDF button, GitHub repo link
2. **Data Sources** — 4 colour-coded cards: MTBS (orange), Landsat/Sentinel-2 (green), Open-Meteo (sky blue), AWS S3/Source Coop (purple)
3. **Methodology Pipeline** — horizontal step-by-step flow: GEE Pull → Cloud Mask → EVI/NDVI → Weather → Analysis
4. **Key Results** — two side-by-side Plotly charts:
   - Burn severity by year (bar chart)
   - EVI recovery timeline comparing low vs high severity (line chart)
5. **Key Findings** — 3 highlighted finding cards:
   - High-severity burn area increased >340% between 1984–2000 and 2001–2022 periods
   - EVI recovery: high-severity areas <60% recovery at 3yr vs >90% for low-severity
   - Weather conditions (>35°C, <15% humidity) correlate with highest severity outcomes
6. **Tools & Links** — tech stack badges (Python, GEE, GeoPandas, Dask, Plotly, Folium, Open-Meteo) + PDF download and GitHub buttons

## Tab 3 — Explorer

**Implementation:** `pages/explorer.py` using pure Streamlit + `streamlit-folium`.

### Layout

**Filter bar (top):**
- Year range slider (1984–2022)
- Severity dropdown (All / Low / Medium / High)
- Minimum acres burned dropdown
- Live matching event count display

**Main area (split layout):**
- Left: Folium map with fire events as colour-coded circle markers
  - Yellow = low severity, Orange = medium, Red = high
  - Dot size proportional to acres burned
  - Pulsing ring on selected event
  - Colour-coded legend (top-right of map)
- Right: Event detail panel (appears on click)
  - Event name + MTBS Event ID
  - 4 stat tiles: acres, severity, temperature, humidity
  - Mini EVI recovery line chart (3yr post-fire)
  - "View full EVI map" link — displays the exported PNG from `data/evi_exports/{event_id}.png` if available; otherwise shows a "no EVI export available" message

**Summary bar (bottom):**
- Live aggregate stats updating with filters: events shown, total acres, % high severity, date range

## Deployment

- Hosted on **Streamlit Cloud** (free tier)
- Repository: GitHub (existing repo)

- Add `.superpowers/` to `.gitignore`

## Out of Scope

- Live GEE queries at runtime
- User authentication
- Data upload functionality
- Mobile-optimised layout (desktop-first for portfolio/research use)
